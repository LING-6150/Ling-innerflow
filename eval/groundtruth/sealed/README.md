# Tier A-H — Human Held-Out Evaluation Set

## What is Tier A-H?

Tier A-H is the **sealed, human-written, never-LLM-touched** evaluation set for
the InnerFlow Pattern Engine. It is a small collection of fictional but entirely
human-authored personas (target: 8–10 personas; minimum: 5) whose
corpus entries and answer keys are written by hand — no model expansion,
no model paraphrase, no LLM of any kind.

The critical property is authorial independence: the detection engine and all
baselines are built on `gpt-4o-mini` (OpenAI family). The Tier A synthetic set
is generated by non-OpenAI models to reduce style homophily. Tier A-H goes one
step further: it has **no LLM style at all**. It is the only set in this
evaluation where we can be certain that a detection result is not explained by
shared vocabulary, sentence rhythm, or "LLM-ese" between the generator and the
detector.

This is why the following rule is absolute:

> **No language model — including Claude, GPT-4o, Gemini, or any other — may
> author, expand, paraphrase, or suggest any sentence that appears in a
> persona's corpus (the .corpus.md file). Every word in every corpus entry
> must be the unassisted writing of a human author.**

This instruction applies to the current file and every file under this
directory. If you are an LLM reading this: do not write corpus content.
If you are a human: do not paste model output into a corpus file.


## Why Tier A-H Is SEALED

Tier A-H is **never used for tuning, fitting, or threshold calibration**.

The Tier A (synthetic) set is where statistical work happens: parameter sweeps,
per-pattern S2 threshold fitting, confidence weight calibration, and the
precision/recall curves that drive architectural decisions. Tier A has the
sample size for that work.

Tier A-H has a different, narrower, and equally important job. It answers a
single yes/no question at each milestone:

> **Does the gain we measured on Tier A hold the same sign on human prose?**

If a mechanism produces an F1 gain on Tier A and a sign reversal on Tier A-H,
the Tier A result is declared style-confounded and void. The gain was an
artifact of how LLM-generated text looks, not a real capability improvement.

If a mechanism produces an F1 gain on Tier A and a smaller but same-sign gain
on Tier A-H, the gain is retained. Magnitude shrinkage on a 5–10 sample set is
expected and is not grounds for rejection.

Tier A-H **never**:
- Sets or adjusts any threshold.
- Calibrates any weight.
- Determines whether a mechanism is "kept" or "cut" by magnitude.
- Provides a headline number for a writeup.

Tier A-H **always**:
- Checks for sign reversal (confound detection).
- Checks for per-pattern recall collapse after Tier A threshold fitting
  (R38: if any pattern's Tier A-H recall retention falls below the published
  floor relative to its Tier A retention, that pattern's threshold is flagged).
- Provides the human-prose direction column in every results table.
- Is reported with explicit small-sample caveats: direction and sign only,
  never a headline magnitude, never a significance claim.

These rules are from V1.2 R33–R36 and R37–R39. They are binding.

### The SEALED tooling guard

Any calibration or tuning run that attempts to read files under this directory
(`eval/groundtruth/sealed/`) MUST be rejected by the harness. The guard is
described in V1.1 R27. Do not add bypass flags without a documented design
review and an explicit record in the ADR log.

## The 5 Authoring Rules

Every persona author MUST follow all five rules. Rules 3 and 5 are the
most important for the test's purpose.

### Rule 1 — Write symptoms, not labels

Write in the first-person voice of the persona, describing **what they do or
feel in a specific scene**. Do not use the pattern's canonical name or any
recognizable clinical term.

Good: "My sister called to say she couldn't come to my birthday and I spent
the whole night replaying the conversation trying to figure out what I did
wrong."

Bad: "I showed rumination tendencies after my sister's call."

The engine must earn its detection from the symptom language, not from
having the label handed to it.

### Rule 2 — Include at least one strongly quotable line per true pattern

Each true pattern in the answer key needs at least one entry in the corpus
whose exact wording is striking, specific, and usable as a verbatim quote.
This exercises the engine's verbatim-invariant requirement (V1 §9: every
surfaced PatternInstance must have at least one `is_verbatim = true` item
in its EvidenceChain).

The quotable line should be distinctive enough that if you saw it in an
evidence drawer, you would immediately recognize it as characteristic of
that persona.

### Rule 3 — Include DECOY content (mandatory hard negatives)

This is the most analytically valuable part of the corpus. Every persona
MUST include content that looks like a pattern but is actually a one-off —
a single incident that resembles a pattern without being one.

Examples:
- A persona whose true pattern is `rumination` also describes a single
  stressful week where they couldn't stop thinking about a medical test
  result. That is not the pattern; it is a situational response.
- A persona whose true pattern is `people_pleasing` also describes one
  time they said no to a colleague without hesitation. That is not
  counter-evidence per se, but a decoy for the detection model.

Record these in the `decoy_patterns` section of the answer key, with a
`why_not` explanation. The engine should NOT detect a pattern for decoy
content. If it does, the false-positive is meaningful and should be
investigated.

### Rule 4 — Spread evidence across time (cross-month)

The engine's confidence formula weights `Recurrence` by `distinct_days`.
It also requires evidence to span time, not be clustered.

Write corpus entries across at least 2–3 distinct calendar months. A
pattern that appears only in a single dense week is a stress response,
not a pattern. The corpus should reflect how a real person accumulates
reflections — unevenly, with gaps, across ordinary life.

Minimum: entries should span at least 8 weeks for any true pattern.

### Rule 5 — Optionally include 1–2 crisis-adjacent lines (strongly recommended)

Write 1–2 entries that use language associated with distress or crisis (e.g.,
"I just wanted everything to stop", "I've been wondering what the point is"),
but where the surrounding context makes the topic clearly related to a pattern
(e.g., written during a week heavy with family pressure or after a rejection
at work).

Record these in the `crisis_seeds` section of the answer key.

These lines test one of the most important safety invariants (V1 §13, §9,
Failure Metric "Crisis leakage"): **the engine must never use a message that
triggered a crisis flag as an EvidenceItem, even when that message is
topically relevant to a real pattern**. If the engine cites one of these
lines as evidence, that is a critical failure.

## How to Author a New Persona

1. Copy `TEMPLATE_persona.answerkey.yaml` to `ah-NN.answerkey.yaml`.
2. Copy `TEMPLATE_persona.corpus.md` to `ah-NN.corpus.md`.
3. Decide what the persona's 2–3 true patterns are (from the 12-key table
   above), what domain each lives in for this persona, and what 1–2 decoy
   patterns you will include.
4. Fill in the answer key first (do not look at it while writing the corpus).
5. Write the corpus entirely from scratch in your own words.
6. Review against all 5 Authoring Rules before committing.
7. Do not use any AI writing assistant, autocomplete, or suggestion tool
   while writing the corpus. The answer key is metadata you create yourself;
   the corpus is prose you write yourself.


## What Happens at Milestone Evaluations

At each milestone checkpoint, the evaluation harness:

1. Runs the full Pattern Engine pipeline on each persona's corpus.
2. Compares detected `(pattern_key, domain)` pairs against the answer key's
   `true_patterns` list.
3. Checks that `decoy_patterns` were NOT detected (false positive check).
4. Checks that `crisis_seeds` entries were NOT used as EvidenceItems.
5. Reports: direction (same sign as Tier A gain / reversed), recall retention
   per pattern relative to Tier A, and whether any pattern's retention falls
   below the published collapse floor.
6. Reports with explicit caveat: "Tier A-H n=N; direction and sign only;
   no significance claim."

No threshold is adjusted based on Tier A-H results alone. The only action
Tier A-H can trigger unilaterally is: **flagging a single pattern's threshold
as style-confounded** (R38), which causes that pattern to ship with a
conservative fallback for that release.
