## 0. Scope of This Amendment

This is a surgical fix in response to the 4.8 engineering review. It addresses
**only the P0 (ship-blocking) findings**, organized under the four areas
requested:

1. Evaluation circularity
2. Retrieval strategy
3. Ground truth design
4. Baseline design

Explicitly **out of scope** for this document (unchanged from V1, by instruction):
- All P1 findings (determinism metric, min-distinct-days, window length, taxonomy
  overlap dedup, κ gate level, ablation significance, cost/incremental refresh).
- All P2 findings.
- The product spec. No product, UI, journeys, or business is touched.

The single root cause behind every P0:

> **The engine is evaluated against data the engine's own model family generated,
> and its decision thresholds are calibrated on a label signal the engine itself
> truncates.** That is "set the exam, write the answer key, and grade yourself."
> V1.1 breaks every link in that loop.

Each section below is a **replacement** for the corresponding part of V1.

## 1. Evaluation Circularity (fixes P0-1, P0-3, P0-4)

Three independent circular dependencies existed in V1. All three are cut.

### 1.1 Generation/detection same-model circularity (P0-1)

**The defect:** V1 §7.3 generated Tier A personas with an LLM; V1 §1.5/§4
detect with `gpt-4o-mini`; V1 §9 baselines are the same model. A detector
recognizing text written by its own model family measures *style homophily*, not
pattern-recognition skill. Any F1 so produced is suspect.

**The fix — strict generator/detector model separation:**

- **R1. The persona generator MUST NOT belong to the detector's model family.**
  The detector is `gpt-4o-mini` (OpenAI). Therefore Tier A synthetic personas are
  generated by a **non-OpenAI** model (e.g. Claude). The generator model id is
  recorded per persona as `generator_model` and is a first-class eval dimension.
- **R2. Cross-model generation panel.** To avoid trading OpenAI-homophily for
  Anthropic-homophily, Tier A is generated by **≥2 distinct non-OpenAI model
  families**, and personas are balanced across them. We then report detection
  metrics **sliced by `generator_model`**. If detector F1 varies by more than a
  set delta across generators, the remaining gap *is* style sensitivity, and we
  report it as such rather than hiding it in an average.
- **R3. No model may be both generator and detector in the same eval run.** This
  is asserted in the harness: `assert generator_model.family != detector.family`.

This converts "can the LLM re-recognize its own writing" into "can an OpenAI
detector recognize patterns written in *another* model's style" — a strictly
harder and more honest question.

### 1.2 Human held-out set — the only style-agnostic ground (P0-1)

Cross-model generation reduces homophily but does not eliminate "LLM-ese." The
only truly style-independent evidence is human-written text.

- **R4. A human-written held-out set is mandatory, not optional.** A small set
  (target 8–10 personas, minimum 5) is **written by hand** — real human prose,
  not model-expanded — with designer-known patterns. This is `Tier A-H`.
- **R5. `Tier A-H` is sealed for tuning.** No threshold, weight, or prompt is
  ever fit on it. It is read **only** at milestone checkpoints, report-only. It
  is the closest thing the project has to a real-distribution test before live
  users, and burning it on tuning destroys its value.
- **R6. The headline honesty metric is the synthetic↔human gap.** We report
  detection metrics on Tier A (LLM-generated) **and** Tier A-H (human) side by
  side. **A large gap invalidates Tier A conclusions.** This gap, not the raw
  Tier A F1, is the number we lead with in any writeup — it is the direct answer
  to "how do you know you're not just measuring style?"

### 1.3 Tier-gap reporting becomes a first-class output (P0-1)

Replaces V1 §8's single aggregate posture.

- **R7.** Every Layer-1/Layer-2 metric is reported as a **triple**: `Tier A
  (synthetic) | Tier A-H (human held-out) | Tier B (real, qualitative)`.
- **R8.** If `metric(Tier A) − metric(Tier A-H)` exceeds the published delta, the
  Tier A result is annotated **"style-confounded, not load-bearing,"** and no
  design decision may cite Tier A alone for that metric.

### 1.4 τ_surface selection-bias loop (P0-3)

**The defect:** V1 §5.4 calibrated `τ_surface` on "surfaced ⇒ confirmed," and V1
§8.1 used confirm-rate (Tier C) as a core Layer-1 metric. But only *surfaced*
candidates can ever be confirmed (product is pull-only), so the system **never
observes the false negatives below τ**. Recall computed this way is
systematically inflated — the classic feedback-loop / selection-bias trap.

**The fix:**

- **R9. τ_surface is calibrated ONLY on Tier A / Tier A-H**, which have complete
  positive **and** negative ground truth, including the sub-threshold region.
  Calibration sweeps τ across candidates the engine computed *before* the
  surfacing cut — so the precision/recall curve sees the candidates that would
  have been hidden. This is the only place the recall side of the knee is real.
- **R10. Tier C (confirm log) is demoted to a monitoring signal.** It is removed
  from the Layer-1 core metric table. It may be tracked over time for drift
  detection, but it **may never be used to calibrate τ, weights, or any
  threshold**, and it is never reported as a primary quality number. (This
  reverses V1 §8.1, which listed recognition-rate as a Layer-1 core metric.)
- **R11. Recall is only ever quoted from a set with observed negatives** (Tier A
  / A-H). Any recall figure sourced from confirm-logs is forbidden in reporting.

### 1.5 ECE/τ shared-signal loop (P0-3)

- **R12.** Confidence calibration (ECE, V1 §5.4) and τ selection use the **same
  Tier A/A-H labels but must not be evaluated on the same split.** The eval set
  is partitioned: a **calibration split** (fit weights + τ + reliability curve)
  and a **sealed report split** (Tier A-H, R5). ECE and the precision/recall knee
  are *reported* on the sealed split, *fit* on the calibration split. Fitting and
  reporting on one split is the circularity; the split removes it.

### 1.6 Verifier circularity — LLM grading LLM (P0-4)

**The defect:** V1 §4.1 used `gpt-4o-mini` to decide `supports` and assign
`strength` 0/1/2, and `strength` fed the confidence formula at weight 0.30 — the
second-largest term. V1 §5 simultaneously declared "we do not trust LLM
self-confidence because it is uncalibrated," then trusted an LLM's `strength`
score, which is uncalibrated in exactly the same way. The verifier itself had no
ground truth.

**The fix:**

- **R13. `strength` is removed from the confidence formula.** It was an
  uncalibrated LLM scalar masquerading as a signal. See §1.7 below for the
  re-weighting. (`strength` may still be *recorded* for analysis, but it does not
  enter `confidence`.)
- **R14. The verifier gets its own ground truth.** A human-annotated subset of
  `(doc, pattern_key)` pairs labeled `supports: yes/no` is built (drawn from
  Tier A-H + Tier B). We report the **verifier's own precision/recall** against
  these human labels. "Citation-grade" is no longer an adjective — it is a
  measured number. Target is published with the verifier, not assumed.
- **R15. Verifier evaluated with a non-OpenAI cross-check.** A second,
  non-OpenAI judge re-labels a sample of `supports` decisions. Agreement between
  the OpenAI verifier and the cross-model judge bounds how much the `supports`
  signal is OpenAI-specific. Disagreements are routed to human adjudication and
  fold back into the R14 set.
- **R16. Batched verification position-bias is measured and controlled.** V1 §4.1
  batched all ≤7 items for a pattern in one call, which invites position bias and
  cross-contamination (early "supports" dragging later items positive), inflating
  evidence counts and the §4.2 "≥3" gate. V1.1 requires:
  - **Order-randomization test:** re-run each batch with shuffled item order; if a
    `supports` decision flips with order, it is unstable and the item is dropped
    (conservative — an unstable citation is not citation-grade).
  - Position-flip rate is a reported eval metric.
  - If flip rate exceeds the published bound, the verifier falls back to
    **independent single-item calls** (more expensive, no cross-contamination).
    The choice between batched and single-item is thus *data-driven*, not assumed.

### 1.7 Confidence re-weighting after removing `strength` (consequence of R13)

The V1 §5.1 weights summed to 1 with `strength` at 0.30. Removing it, the three
remaining **observable, non-LLM-scalar** factors are renormalized:

```
confidence = w_e·Evidence + w_r·Recurrence + w_t·Recency
Evidence    = min(1.0, verified_item_count / 5)
Recurrence  = min(1.0, distinct_days / 4)
Recency     = exp(−daysSinceLatestItem / 90)

w_e = 0.50, w_r = 0.30, w_t = 0.20   (calibration starting point, see R17)
```

Every term is now a **count or a time function** — fully derivable from the
EvidenceChain, zero LLM-subjective scalars. This finally makes good on V1 §5's
own stated principle. Final weights are a calibration output (R17), not these
literals.

## 2. Retrieval Strategy (fixes P0-2)

### 2.1 Irreversible recall loss at S2 — the core defect

**The defect:** V1 §1.5 candidate recall used a `probe vector = mean(neutral_
description + evidence_shapes)` — an **abstract** vector — to retrieve against
**concrete, symptom-shaped** user utterances. V1 §3.1 itself acknowledges this
mismatch ("users don't write 'I have people-pleasing tendencies'") and solves it
with HyDE *at S3*. But S2 runs *before* S3, so the abstract probe
systematically under-recalls symptom-style language at the candidacy gate. **You
cannot recover at S3/S4 what you failed to recall at S2.** Recall loss is
irreversible; the precision stages downstream only ever subtract.

### 2.2 Fix: S2 uses HyDE exemplar vectors, not abstract probes

- **R17-RET (replaces V1 §1.5 embedding recall):** S2 embedding recall queries
  with the **same pattern-conditioned HyDE exemplar vectors defined in V1 §3.1**
  (the 3 first-person symptom utterances), not the abstract probe. The recall
  query and the evidence query now live in the **same symptom-shaped vector
  space**. The cheap recall stage and the precise evidence stage stop
  contradicting each other.
- The abstract probe vector is **retired** as a recall mechanism. (It may remain
  only as a coarse tie-breaker, never as a gate.)
- LLM recall (V1 §1.5 signal 2) is unchanged; union semantics unchanged.

### 2.3 Stacked recall gates: bound the recall ceiling, don't multiply it blindly

**The defect:** V1 stacked three uncalibrated recall-killing gates —
`min-corpus-docs=20` (§1.4), `max-sim ≥ 0.78` (§1.5), per-candidate `≥5 docs
above 0.5` (§3.4) — whose pass-rates multiply, crushing the recall ceiling the
product most fears losing.

**The fix:**

- **R18. The product cares most about recall; therefore recall gates are
  recall-tuned, not precision-tuned.** Where two stages both protect precision,
  the *earlier, blunter* one (the S2 similarity gate) is loosened and precision is
  recovered by the *later, smarter* stages (entailment verification, confidence
  τ). Concretely: the S2 candidacy gate is set to a high-recall operating point;
  it exists to drop only the clearly-absent, not to be a precision filter.
- **R19. The recall ceiling is measured, not assumed.** The harness reports
  **stage-wise recall retention** on Tier A/A-H: of the truly-present patterns,
  what fraction survives S1, S2, S3.4, S4.2 respectively. This exposes exactly
  which gate is the recall bottleneck instead of leaving it implicit in a product
  of three magic numbers.

### 2.4 Every threshold is a calibration output, not a chosen constant

**The defect:** `0.78`, `0.5`, `20`, `5`, and `min-distinct-days=2` appeared in
V1 as bare literals — magic numbers with no derivation, each applied uniformly
across 12 patterns with different vector distributions.

**The fix:**

- **R20. All retrieval thresholds are declared "calibration outputs, not
  chosen."** In V1.1 they are written as named parameters with a *search range*,
  and their values are produced by the §1.4/§1.5 calibration sweep on Tier A/A-H,
  reported with the value that was selected and the metric it optimized:

  | Parameter | V1 literal | V1.1 status |
  |---|---|---|
  | `gate.min-corpus-docs` | 20 | calibration output, range [10,40] |
  | `recall.s2-sim-threshold` | 0.78 | calibration output, **per-pattern**, range [0.60,0.85] |
  | `candidate.support-min-docs` | 5 | calibration output, range [3,8] |
  | `candidate.support-min-relevance` | 0.5 | calibration output, range [0.4,0.7] |

  (`min-distinct-days` is a P1 item and is left unchanged per scope.)
- **R21. The S2 similarity threshold is per-pattern, not global.** Because ada-002
  cosine distributions differ across patterns, a single global cutoff is
  necessarily too loose for some and too strict for others. Each `pattern_key`
  gets its own calibrated S2 cutoff, fit on that pattern's positive/negative
  examples in Tier A/A-H. A global constant is explicitly rejected.

## 3. Ground Truth Design (fixes P0-1, P0-4)

Replaces V1 §7 where it conflicts.

### 3.1 Tiering, corrected

| Tier | Source | Generator constraint | Role |
|---|---|---|---|
| **A** | Synthetic personas | **non-OpenAI**, ≥2 model families (R1–R3) | Tunable known-answer set with full negatives |
| **A-H** | **Human-written** personas (≥5, target 8–10) | no model (R4) | **Sealed** style-agnostic report set (R5) |
| **B** | Real opt-in users (3–5) | n/a | **Qualitative** sanity check only (see R24) |
| **C** | Live confirm/reject log | n/a | **Monitoring only**, never calibration (R10) |

### 3.2 Hard negatives and cross-model generation are mandatory (P0-1)

- **R22.** Tier A retains V1 §7.3's mandatory decoys/hard-negatives, **and** every
  persona now carries `generator_model`. Personas are balanced across generator
  families so that "detectable" cannot mean "written by the detector's family."
- **R23.** The persona *spec* (intended patterns, decoys, crisis seeds) remains
  the authority; only the surface text is model-expanded — and now by a foreign
  model. For Tier A-H, even the surface text is human-authored.

### 3.3 Tier B is explicitly qualitative (P0-1 honesty)

- **R24.** With 3–5 real users, dual annotation + adjudication cannot yield
  statistical conclusions. Tier B is declared a **qualitative ecological sanity
  check**, not a quantitative eval source. No precision/recall/F1 headline is ever
  computed *primarily* on Tier B; it is used to ask "does anything here look
  obviously wrong on real prose?" and to populate the verifier's human-label set
  (R14).

### 3.4 Ground truth for the verifier (P0-4)

- **R25.** Ground truth is no longer only at `(user, pattern_key, domain)`
  granularity. A second ground-truth layer is added at **`(doc, pattern_key)`
  entailment granularity**, human-labeled `supports: yes/no` (R14), drawn from
  Tier A-H and Tier B. This is what makes the verifier (§1.6) measurable rather
  than asserted.
- **R26.** This entailment set is also the substrate for the position-bias test
  (R16) and the cross-model verifier check (R15).

### 3.5 Storage & sealing

- **R27.** `eval/groundtruth/` holds Tier A + Tier A-H + the entailment set
  (synthetic and human-authored text is in-repo; **no real Tier B user text in
  git** — IDs only, per V1 §7.5). Tier A-H lives under `eval/groundtruth/sealed/`
  with a tooling guard that fails any run attempting to read it during a
  tuning/calibration job (enforces R5).

## 4. Baseline Design (fixes P0-1)

Replaces V1 §9 where it conflicts. (V1's baseline *ladder* B0→B3 + ablation table
is sound and retained; the P0 fix is about **what model the baselines run** and
**what they're scored against**.)

### 4.1 Baselines must not inherit the detector's homophily (P0-1)

**The defect:** V1's B2 (single-prompt LLM) and B3 (retrieval-no-verification)
used the **same `gpt-4o-mini`** as the full system. If both the baseline and the
full system enjoy style-homophily with LLM-generated Tier A, the *gap* between
them still looks meaningful while both are inflated — and a reviewer cannot tell
how much of "full beats B2" is real capability vs shared homophily.

**The fix:**

- **R28. Every baseline is scored on the same triple as the full system** — Tier
  A | Tier A-H | Tier B (R7). The number that matters for "did our pipeline
  help" is the **gap on Tier A-H (human, sealed)**, not the gap on Tier A. A gain
  that exists on Tier A but vanishes on Tier A-H is style-confound, not
  engineering value, and is reported as such.
- **R29. B2/B3 run under the cross-model honesty check.** Because Tier A is
  generated by non-OpenAI models (R1), B2/B3 (OpenAI) are already not scoring
  against their own model's text — this is inherited automatically once §3 is in
  place. The harness asserts it (`baseline.model.family != generator.family`).
- **R30. A non-LLM baseline carries the floor.** B0 (prior) and B1 (BM25/lexical)
  remain — and gain importance, because they are **immune to LLM homophily
  entirely.** If the full LLM pipeline cannot beat B1 *on Tier A-H*, the LLM
  machinery is not earning its cost on style-independent data. B1 is the honest
  floor; the headline claim becomes "full beats B1 on human-written held-out
  data," which no amount of generation homophily can fake.

### 4.2 The verification gap is attributed cleanly (consequence of P0-4)

- **R31.** To support V1 §9.4's claim that B3→full measures "the value of
  verification," V1.1 requires the B3→full comparison to be run as a **single-
  variable ablation** (add only §4 verification, holding domain-reassignment,
  grounded-summary, and τ constant) rather than as a multi-variable jump. (The
  statistical-significance dimension of this is a P1 item and is left to V1's
  §9.5/§9.6 per scope.)

### 4.3 Reporting

- **R32.** The V1 §9.6 results table gains a mandatory column structure: each
  cell is the `Tier A | Tier A-H` pair. Any design decision (keep/cut a
  mechanism) **must be defensible by the Tier A-H column**, not Tier A. Tier A
  alone is never sufficient grounds for an architectural decision.
